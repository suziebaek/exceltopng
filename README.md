# Cell 통합 카드 이미지 생성기 (Streamlit)

엑셀 파일을 업로드하면, `챕터 + Cell (개념)` 단위로 **Rule Reminder / Error Spotlight / Practice**
세 가지 내용을 하나로 이어붙인 이미지를 Cell마다 1장씩 만들어줍니다.

## 엑셀 파일 요구 조건
시트 3개(이름에 아래 키워드가 포함되어 있으면 자동으로 인식합니다):
- `Rule Reminder ...` — 컬럼: 챕터 / Cell (개념) / 구성 요소 / 내용
- `Error Spotlight ...` — 컬럼: 챕터 / Cell (개념) / 오류 패턴 / 틀린 문장 / 올바른 문장 / 이유
- `... Practice ...` — 컬럼: 챕터 / Cell (개념) / 유형 / 내용 / 정답 및 해설

각 시트의 1행은 제목(병합 셀), 2행이 실제 컬럼 헤더입니다. `챕터`, `Cell (개념)` 컬럼은
병합 셀로 비어있는 행이 있어도 자동으로 위쪽 값을 채워서(forward-fill) 처리합니다.

## 파일 구성
```
grammar_app/
├── app.py            # Streamlit 앱
├── sample_input.xlsx # 테스트용 예시 엑셀 (CH1~3 문법정리)
├── requirements.txt
└── fonts/
    ├── Know_Regular.ttf   # 아는체 Regular - 섹션 배너(제목)용
    ├── Know_Bold.ttf      # 아는체 Bold   - 섹션 배너(제목)용
    ├── Do_Light.ttf       # 하는체 Light  - 본문용 (현재 미사용, 필요 시 활용)
    ├── Do_Regular.ttf     # 하는체 Regular - 표 본문용
    ├── Do_Bold.ttf        # 하는체 Bold   - 표 헤더용
    └── NotoFallback-Regular.ttf  # 특수문자 안전망 폴백 폰트 (경량 서브셋)
```

## 실행 방법
```bash
pip install -r requirements.txt
streamlit run app.py
```
브라우저가 열리면 엑셀 파일을 업로드하세요. 업로드 후:
- 챕터별 탭으로 나뉘어 각 Cell의 통합 이미지가 펼쳐서 보여집니다.
- 각 이미지 아래 `⬇ 다운로드` 버튼으로 개별 PNG를 받을 수 있습니다.
- 맨 아래 `⬇ 전체 이미지 ZIP으로 다운로드`로 모든 Cell 이미지를 한 번에 받을 수 있습니다.

## 특수문자 처리
아는체/하는체 폰트에는 `⚠`, `✓`, `✗`, `∙` 같은 일부 특수문자 글리프가 없어서 빈 네모(□)로
깨집니다. 이를 위해 두 단계로 대응합니다.
1. **알려진 치환표** (`SYMBOL_REPLACEMENTS`): `⚠→※`, `✓→(O)`, `✗→(X)`, `∙→·` 등 확인된 문자를
   폰트에 있는 안전한 문자로 자동 치환합니다.
2. **안전망 폴백 폰트** (`fonts/NotoFallback-Regular.ttf`, Noto Sans CJK에서 기호 영역만 추출한
   경량 서브셋): 치환표에도 없는 미지의 문자가 나오면 이 폴백 폰트로 그 글자만 대신 그려서
   최소한 깨지지 않게 합니다.

새로운 특수문자가 계속 깨진다면 `app.py`의 `SYMBOL_REPLACEMENTS` 딕셔너리에 항목을 추가하세요.

## Practice 정답 가리기 (2가지 버전)
Cell마다 이미지가 **2장** 생성됩니다.
- `..._정답.png` — Rule Reminder / Error Spotlight / Practice(정답·해설 포함) 전체
- `..._문제.png` — 위와 동일하지만 Practice 표의 "정답 및 해설" 칸만 `정답은?` 문구로 가린
  빈칸 연습용 버전 (컬럼명에 "정답"이 포함된 컬럼을 자동으로 찾아서 가림)

두 버전 모두 화면에 나란히 표시되고, 개별 다운로드 버튼과 전체 ZIP에 함께 포함됩니다.

## 파일명 규칙
다운로드되는 이미지 파일명은 다음 형식을 따릅니다.
```
BOOST-GR-{레벨}-CH{챕터번호 2자리}-C{Cell번호}-{1 또는 2}.png
```
- `BOOST`, `GR` : 고정값
- `{레벨}` : 업로드한 엑셀 **파일명**에서 자동으로 찾은 `H` 또는 `E` (예: `..._H.xlsx` → `H`).
  못 찾으면 화면 상단의 "레벨 코드" 입력창에 직접 입력할 수 있습니다 (입력 전엔 `X`로 표시).
- `CH{챕터번호}` : 챕터 텍스트에서 숫자만 추출해 2자리로 0 채움 (예: "CH3. 조동사" → `CH03`)
- `C{Cell번호}` : Cell 텍스트에서 "Cell 뒤 번호"만 추출 (예: "Cell 2. 현재완료" → `C2`)
- 마지막 숫자 : **1 = 빈칸 연습용(정답 가림) 버전, 2 = 정답·해설 포함 버전**

예: `BOOST-GR-H-CH01-C1-1.png` (H 레벨, 1챕터, 1번 셀, 빈칸 연습용),
`BOOST-GR-H-CH01-C1-2.png` (동일 셀의 정답·해설 포함 버전)

## Cell 매칭 방식 (중요)
시트마다 "Cell (개념)" 텍스트가 완전히 똑같지 않을 수 있습니다 (예: Error Spotlight 시트에는
괄호 설명이 빠져 있는 경우). 텍스트를 그대로 비교하면 같은 개념인데도 다른 그룹으로 인식되어
이미지가 쪼개지는 문제가 있었습니다. 이를 방지하기 위해 "Cell (개념)" 텍스트 전체가 아니라
**"Cell 뒤에 오는 번호"만 추출**해서 그룹 키로 사용합니다 (예: "Cell 1. 조동사의 종류와 강도
Ⅰ (의무·허락·요청)"과 "Cell 1. 조동사의 종류와 강도 Ⅰ"은 둘 다 "1"로 인식되어 같은 그룹으로
합쳐짐). 화면/파일명에 쓰이는 대표 텍스트는 세 시트 중 가장 설명이 긴(자세한) 텍스트를
자동으로 채택합니다.

같은 챕터 안에 "Cell 1"이 두 번 이상 등장하면 하나로 합쳐지므로, 각 챕터 안에서 Cell 번호가
겹치지 않도록(1, 2, 3 ...) 엑셀을 구성해 주세요.

## 이미지 구성
한 장의 이미지 안에 위에서부터
1. **Rule Reminder** (연보라 배너) — 해당 Cell의 핵심 규칙 표
2. **Error Spotlight** (연분홍 배너) — 해당 Cell의 오류 패턴 표
3. **Practice** (연초록 배너) — 해당 Cell의 연습 문항 표

가 순서대로 쌓입니다. "Cell 1. 자동사"처럼 셀 이름 자체를 이미지 위에 표시하지는 않으며,
내용(규칙 설명)에서 어떤 개념인지 알 수 있도록 구성했습니다. 대신 다운로드되는 파일명에는
`챕터_Cell명.png` 형태로 구분 정보가 들어갑니다 (예: `CH1_동사의_종류_Cell_1_자동사.png`).

특정 Cell에 Rule/Error/Practice 중 하나라도 내용이 없으면 해당 섹션은 이미지에서 자동으로
제외됩니다.

## 폰트 (중요)
이미지에 들어가는 한글 텍스트는 시스템에 설치된 폰트가 아니라, **앱과 함께 배포되는
`fonts/` 폴더의 파일을 직접 읽어서** 렌더링합니다. 그래서 배포 서버(Streamlit Community
Cloud 등)에 어떤 한글 폰트가 설치되어 있는지와 무관하게 항상 동일하게 동작하고, 이전에 나던
`OSError: cannot open resource` (폰트 파일 경로 없음) 문제가 재발하지 않습니다.

- **아는체(Know)** — 각 섹션 배너("Rule Reminder" / "Error Spotlight" / "Practice") 제목용
- **하는체(Do)** — 표 헤더(Bold) 및 표 본문(Regular) 용

`app.py`를 배포할 때는 반드시 `fonts/` 폴더 전체를 같은 경로 구조로 함께 올려야 합니다.
경로가 하나라도 없으면 해당 폰트는 PIL 기본 폰트로 자동 대체되고(앱이 죽지 않음), 화면에
경고 메시지가 1회 표시됩니다.

폰트 역할을 바꾸거나 추가하고 싶으면 `app.py` 상단의 `FONT_FILES` 딕셔너리를 수정하세요:
```python
FONT_FILES = {
    "title_regular": FONTS_DIR / "Know_Regular.ttf",
    "title_bold": FONTS_DIR / "Know_Bold.ttf",
    "body_light": FONTS_DIR / "Do_Light.ttf",
    "body_regular": FONTS_DIR / "Do_Regular.ttf",
    "body_bold": FONTS_DIR / "Do_Bold.ttf",
}
```

## 다른 교재 파일로 확장하려면
- 시트 이름과 컬럼 이름 규칙(위 "엑셀 파일 요구 조건")을 지키면 어떤 챕터/과목이든 그대로 동작합니다.
- 색상, 폰트 크기, 이미지 너비 등은 `app.py` 상단의 상수(`RULE_COLOR`, `TOTAL_WIDTH`, `LINE_H` 등)에서
  조정할 수 있습니다.
